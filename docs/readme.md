## **✨ 프로젝트 소개**

### 주제

```
AI 서비스 기반 반려동물 쇼핑몰: RAG 챗봇 상담 및 이미지 인식, 화면 인식 등 편의 기능을 제공합니다.
```

&nbsp;

### 기획의도

반려동물 양육 인구가 급증함에 따라 관련 용품 시장도 빠르게 성장하고 있지만, 수많은 상품 정보 속에서 내 반려동물에게 꼭 필요한 제품을 찾기란 쉽지 않습니다. 특히 처음 반려동물을 키우거나 특정 상황(질병, 알레르기 등)에 놓인 보호자들은 어떤 기준으로 상품을 선택해야 할지 막막함을 느끼곤 합니다.

**PetGPT**는 이러한 정보 비대칭 문제를 해결하고, 보호자들이 더욱 편리하고 현명하게 쇼핑할 수 있는 환경을 제공하고자 기획되었습니다. RAG(Retrieval Augmented Generation) 기반 AI 챗봇 상담과 이미지 인식을 통한 품종 분석 등 AI 편의 기능을 통해 고객 맞춤형 쇼핑 경험을 제공합니다.

## **🤔 트러블 슈팅 (Troubleshooting)**

- **김민주**:

  - **문제 1**: 화면에서 이미지가 로드되지 않음
    - **원인**: jQuery의 로딩 시점(선/후)에 따라 스크립트 실행 여부가 달라질 수 있음
    - **해결**: 이미지가 JS(jQuery)로 동적으로 로드되는 구조라면, jQuery가 반드시 먼저 로드되어야 함
  - **문제 2**: 잘 사용하던 annotation이 갑자기 오류를 일으킴
    - **원인**: 캐시삭제가 제대로 되지 않음
    - **해결**: 인텔리제이에서 'invalidate cashes'를 선택해 재시작
  - **문제 3**: 문의글에서 주문내역 선택 상자가 두 개 나타나는 오류 발생
    - **원인**: select 태그 하나에 Select2, NiceSelect가 모두 적용되어 중복으로 나타남
    - **해결**: 해당 셀렉트박스에 niceSelect()가 적용되지 않도록 코드 작성

- **김준서**:

  - **문제 1**: 관리자 페이지에 particles.js를 이용해 파티클이 내리는 효과를 추가하려 했지만, 효과가 화면에 나타나지 않음.
    - **원인**: particles.min.js 스크립트가 로드되지 않음
      CDN 경로 누락 또는 인터넷 연결 문제
      <div id="particles-js">가 페이지에 없음
      JavaScript가 렌더링되기 전에 엘리먼트를 찾지 못함
      CSS 설정 누락으로 입자가 보이지 않음
      z-index, position, width, height 미지정
      스크립트가 <body>보다 먼저 실행됨
      DOM이 준비되기 전에 실행되어 canvas가 생성되지 않음
    - **해결**: 구현하지 못함

- **김상우**: 
    - **문제 1**: 주문상세 페이지 로딩 후 콘솔에 “DataTables warning: table id=totalOrderTable – Incorrect column count” 오류가 발생하여 데이터가 페이지에 출력되지 않는 현상이 발생했다.
        - **원인**: 요약 테이블 아래에 “상세” 토글용 <tr class="detail-row"> 를 JSP에서 그대로 남겨두면, DataTables가 헤더(thead) 열 개수(7개)와 실제 <tbody> 행의 <td> 개수(달라짐)를 비교하면서 경고를 출력하였다.
        - **해결**: 요약 행(<tr>…</tr>)만 남기고, detail-row 관련 <tr> 요소는 모두 지운
          다.
          대신 JavaScript의 row.child() API를 사용해 “클릭 시 동적으로 한 줄(span)처럼 펼쳐지는” 방식으로 변경하여 해결을 하였다.

    - **문제 2**: 관리자 채팅 화면에서 “나에게” 메시지를 보냈을 때 입력된 텍스트는 사라지지만, 채팅 박스(#me 영역)에는 전혀 출력되지 않는다.
      - **원인**: 구독(Subscription) 호출이 잘못된 컨텍스트(context)에서 실행됨
      - **해결**: STOMP 클라이언트 인스턴스를 올바르게 참조하여 구독하여 해결하였다.
- **김현호**:

  - **문제**: AI 챗봇이 사용자의 질문 의도와는 다소 거리가 있거나 관련성이 낮은 정보를 포함하여 답변하는 경우가 발생했습니다. 특히, 쇼핑몰 내부 특정 상품 정보나 정책 관련 질문에 대해 기대만큼 정확한 답변을 제공하지 못했습니다.

    - **원인**: 벡터 스토어에 문서를 저장하고 임베딩할 때 사용한 청크(chunk)의 크기 및 분할 전략이 부적절했습니다. 초기에는 문서의 의미적 단위를 넓게 가져가기 위해 청크를 매우 크게 설정했습니다. 이 경우, 개별 청크가 너무 많은 정보를 포괄하게 되어 검색 시 특정 질문과 관련된 핵심 내용의 정확도가 오히려 떨어지고, 관련 없는 정보까지 함께 참조하여 답변을 생성하는 경향이 있었습니다. 반대로, 정보의 정확도를 높이고자 청크를 문장 단위 등으로 매우 잘게 분할했을 때는 개별 청크가 충분한 컨텍스트를 담지 못했습니다. 이로 인해 여러 청크에 분산된 정보를 종합적으로 이해하고 답변을 생성하는 데 어려움이 있었으며, 단편적이거나 맥락이 부족한 답변이 생성되었습니다.

    - **해결**: 다양한 청크 크기(예: 문서 전체, 문단 단위, 특정 토큰 수 기준, 문장 단위)와 분할 전략(고정 길이, 의미 기반 분할 등)을 실험적으로 적용하며 답변 품질을 지속적으로 비교 평가하여 문제를 해결했습니다.

  나는 우리 팀원들이 자랑스럽고, 감사하다.

- **김민주**: 이번 프로젝트는 처음으로 직접 참여한 프로젝트였기 때문에 초반에는 모든 것이 낯설고 어려웠지만 시간이 지날수록 점점 애정이 생기고 흥미도 커졌습니다. 제가 구상한 대로 웹사이트를 디자인하고 기능을 구현하는 과정은 생각했던 것보다 훨씬 재미있고 보람 있었습니다.
또한 두 달 동안 필요한 기술들을 찾아보고 공부하며 하나씩 기능을 완성해 나가는 과정 자체가 큰 자산이 되었습니다. 이 경험을 통해, 중요한 것은 어떤 기술을 이미 알고 있느냐보다 목표를 이루기 위해 어떻게/무엇을 배우고자 했으며, 그 기술을 얼마나 효율적으로 체득하느냐는 점이라는 것을 깨달았습니다.
마지막으로, 두 달이라는 긴 시간 동안 함께 고민하고 노력해준 팀원들에게 진심으로 감사드립니다. 혼자였다면 결코 완성할 수 없었을 이 프로젝트를 함께 만들어가면서, 협업의 중요성과 서로에 대한 배려, 존중의 가치를 많이 배울 수 있었습니다.

  또한 팀원들과의 협업 과정에서 버전 관리(Git), 이슈 트래킹, 코드 리뷰 등의 개발 문화가 프로젝트 완성도에 직접적인 영향을 미친다는 것을 느꼈습니다. 특히 의사소통과 역할 분담이 원활할수록 개발 속도와 품질이 향상된다는 점을 실감했습니다.

- **김상우**: 프로젝트를 함께해 준 팀원 여러분께 먼저 깊은 감사의 말씀을 전합니다. 이번 첫 번째 프로젝트를 통해 가장 크게 느낀 점은, 목표와 방향을 충분히 공유하지 않으면 서로 다른 기대치로 인해 엇박자가 날 수 있다는 것입니다. 프로젝트 초반에 각자가 생각하는 기대 결과와 진행 방식을 자유롭게 제안하며 의견을 나눴습니다. 이 과정에서 목표는 같다고 해도 표현 방식이나 우선순위에 차이가 있을 수 있다는 사실을 깨달았고, 서로의 관점을 이해하며 조율하는 자체가 곧 프로젝트의 핵심이라는 것을 체감했습니다.
모두가 자유롭게 키워드와 아이디어를 던지면, 다른 팀원은 장단점을 고민하며 필요한 자료를 공유했습니다. 실시간으로 의견을 주고받으면서 서로의 생각이 어떻게 발전하는지 눈으로 확인할 수 있었고, 덕분에 팀워크가 더욱 단단해졌습니다. 역할을 분명히 분담하되, 서로 돕고 겹치는 부분에 대해서는 언제든 협업할 수 있어야 효율이 극대화된다는 점도 배웠습니다.
또한, 문제 앞에서 포기하지 않는 노력과 적극적인 도움 요청의 중요성도 절실히 느꼈습니다. 오류나 어려움이 생길 때마다 질문을 던지고 도움을 청하자, 경험 있는 팀원이 상세한 조언과 자료를 아낌없이 공유해 주었습니다. 서로의 약점을 보완해 주고, 어려울 때는 주저 없이 손을 내밀 수 있는 신뢰가 있기에 문제를 빠르게 해결할 수 있었습니다.
결국 이번 프로젝트를 통해 확인한 가장 큰 교훈은 ‘협업’과 ‘노력’이 서로 시너지를 낼 때야 비로소 좋은 성과를 얻을 수 있다는 점입니다. 앞으로도 어떤 프로젝트를 맡든, 지속적으로 의견을 나누고 서로의 부족한 부분을 메워 주는 협업 정신을 한층 더 발전시켜 나가겠습니다.
